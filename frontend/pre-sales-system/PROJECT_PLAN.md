# 售前流程全生命周期管理系统 - 项目规划

## 项目概述

**项目名称**: 售前流程全生命周期管理系统  
**定位**: 面向售前解决方案工程师的专业工作平台  
**目标**: 实现售前项目从线索到签约的全流程数字化、标准化管理

## 核心业务流程

### 1. 线索管理阶段
- 线索录入与分类
- 线索评估与分级
- 线索分配与跟进
- 线索转化为商机

### 2. 商机管理阶段
- 商机信息完善
- 需求分析与记录
- 技术方案制定
- 投标项目管理

### 3. 解决方案阶段
- 方案文档管理
- 方案版本控制
- 方案评审流程
- 方案审批跟踪

### 4. 投标管理阶段
- 标书制作协调
- 投标进度跟踪
- 价格策略管理
- 投标结果记录

### 5. 合同管理阶段
- 合同起草
- 合同评审
- 合同谈判记录
- 签约状态跟踪

## 系统功能模块

## 技术栈选择

### 前端技术
- **框架**: React 18 + TypeScript
- **UI组件库**: Ant Design Pro 5.x
- **状态管理**: Redux Toolkit + RTK Query
- **路由管理**: React Router v6
- **表单处理**: React Hook Form + Zod
- **图表可视化**: ECharts + @ant-design/charts
- **富文本编辑器**: TinyMCE / Quill

### 后端技术
- **语言**: Node.js + TypeScript
- **框架**: NestJS 10.x
- **数据库**: PostgreSQL (主库) + Redis (缓存)
- **ORM**: Prisma
- **认证授权**: JWT + Passport.js
- **文档管理**: MinIO / AWS S3
- **全文搜索**: Elasticsearch

### DevOps
- **容器化**: Docker + Docker Compose
- **CI/CD**: GitHub Actions
- **监控**: Prometheus + Grafana
- **日志**: ELK Stack

## 数据库设计核心实体

### 核心实体关系
```
User (用户) → Project (项目) → Opportunity (商机)
                         ↓
                    Solution (解决方案) → Document (文档)
                         ↓
                    Tender (投标) → Contract (合同)
```

### 主要数据表
1. **users**: 用户表
2. **projects**: 项目表
3. **opportunities**: 商机表
4. **solutions**: 解决方案表
5. **documents**: 文档表
6. **tenders**: 投标表
7. **contracts**: 合同表
8. **workflows**: 工作流表
9. **knowledge_base**: 知识库表
