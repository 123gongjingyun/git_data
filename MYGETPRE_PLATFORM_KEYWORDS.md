# MYGETPRE 平台效果关键词文档

## 1. 用途

这份文档用于把当前平台已经具备的页面效果、业务能力、权限结构、审批链路、飞书/OpenClaw 集成、部署约束和运维口径压缩为一份可直接复用的高密度关键词说明。

目标：

- 拿着这份文档，可以快速向模型说明“这个平台是什么、长什么样、有哪些模块、模块之间如何联动”；
- 作为后续多 agent 拆单、OpenClaw 联调页优化、前端重构、演示页生成、设计复刻的统一提示词基线；
- 避免后续会话只记住局部页面，而忘记平台整体结构、权限口径、部署边界和真实集成能力。

适用场景：

- 生成平台级页面效果说明；
- 生成新会话的项目背景提示词；
- 拆分多 agent 任务前的统一上下文；
- 规划 OpenClaw 联调页优化时的边界约束；
- 复刻整个售前管理平台的 UI / 功能全景。

## 2. 一句话总描述

企业级售前流程全生命周期管理平台，作为 CRM 增强模块，覆盖线索、商机、解决方案、投标、合同、知识库、工作台、数据分析、权限中心、审批流程库、飞书集成和 OpenClaw 只读联调，采用 React + TypeScript + Vite + Ant Design 前端，NestJS + TypeORM + MySQL 后端，支持本地优先验证、云上 Docker Compose 轻量部署、多 agent 并行协作和 GitHub 检查点管理。
